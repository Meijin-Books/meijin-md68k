# 第二部　OS実装編 ── MeijinOS68k

『MeijinOS68k』（本書 第13〜28章）のソースコードです。68000（メガドライブ）の上で動く、
小さなマルチタスクOSの全実装。第一部で学んだ命令だけを使い、新しいCPUの知識は一つも増えません。

心臓部（コンテキストスイッチ）は **13命令**。1ページに印刷して、全行を説明できます（第16章）。

```asm
Yield:
    ori.w   #$0700,sr           ; ⓪ 割り込みを止める（切り替え中は触らせない）
    movem.l d0-d7/a0-a6,-(sp)   ; ① 世界を封印
    lea     TaskTable,a0        ; ② 台帳へ、いまのSPを記録
    move.w  CurTask,d1
    move.w  d1,d0
    lsl.w   #3,d0
    move.l  sp,(a0,d0.w)
    bsr     SelectNext          ; ③ 次を選ぶ
    move.w  d0,CurTask          ; ④ 台帳から引く
    lsl.w   #3,d0
    move.l  (a0,d0.w),sp        ; ⑤ SPを差し替える ＝ 世界の切り替え
    movem.l (sp)+,d0-d7/a0-a6   ; ⑥ 世界を復元
    rte                         ; ⑦ 向こうの世界へ帰る
```

⓪ は、68000の trap が割り込みマスクを上げないための備えです。
これが無いと、切り替えの最中にVBlankが割り込んで台帳が壊れます
（数万回に一度しか起きない、再現しないフリーズの正体）。

## ビルド

必要なのは [vasm](http://sun.hasenbraten.de/vasm/)（m68k / Motorola syntax）だけです。

```sh
make            # 全部ビルド
make coop.bin   # 一つだけ
```

Windows で `make` が無い場合は、`build.bat` を使ってください。

```
build.bat          全部ビルド
build.bat coop     一つだけ
build.bat clean    .bin を消す
```

どちらも使わず、手で叩く場合：

```sh
vasmm68k_mot -Fbin -o coop.bin coop.asm
```

できた `.bin` を [BlastEm](https://www.retrodev.com/blastem/) などの
メガドライブ・エミュレータに読み込ませてください。実機でも動きます。

## 動かすと、何が見えるか

**`coop.asm`** — 上下の行に、同じ値が並んで増えていきます（`2A = 2A`）。
別々のタスクが、別々のスタックで、互いを知らずに数えている。
それが同じ値になるのは、きっちり交代で動いている証拠です。

**`preempt.asm`** — どちらのタスクも `trap #0` を呼びません。
それでも両方が動きます。OSが時間を取り上げているからです。

**`meijin10.asm`** — 三人の住人が同居します。時計（1秒に1つ）、
Aボタンのカウンタ、そして譲らないゾンビ。
ゾンビが暴れても、他の二人は平気で動き続けます。

**`crash.asm`** — タスク0が配列の大きさを間違え、隣のタスク1を踏み潰します。
タスク0に悪意はなく、タスク1に落ち度もない。それでもタスク1は死に、
CPUは何も言いません。これが「保護が半分しかない」ということです。

**`trace.asm`** — タスク1のSRのTビットを立てると、1命令ごとに例外が起きます。
実行した命令の数と、最後の番地が画面に出ます。
これが、デバッガの「ステップ実行」の正体です。

## 検証

本文の主張が本当かどうかを、自動で確かめられます。

```sh
make verify
# または
python3 tools/verify.py
```

`tools/emu68k.py` は、この検証のために書いた小さな68000エミュレータです
（本書のコードが使う命令だけを実装しています）。
BlastEm の代わりにはなりませんが、
「タスクが切り替わっているか」「特権違反を捕まえられるか」
「隣人が本当に死ぬか」を、機械的に確認できます。

```
[3] 第16章：心臓部は、何命令か
  [OK ] Yield は 13命令（本文の主張）   実測 13 命令
[4] 第17章：協調的マルチタスク（2A = 2A）
  [OK ] 2つのカウンタは、ほぼ揃う（差は0か1）   差=1
[7] 第23章：特権違反を、捕まえる
  [OK ] 特権命令を使ったタスクだけが、殺される   alive=[1, 1, 0]
[9] 第26章：隣人を殺す（保護は半分しかない）
  [OK ] 被害者（タスク1）は、死んだ   何もしていないのに
  ...
結果： 27 / 27 項目が OK
```

> 注：`tools/emu68k.py` と `tools/verify.py` は今回のアップロードに含まれていませんでした。
> 既存の OS リポジトリからここに配置し、`verify.py` の出力メッセージ内の章番号だけ
> 本書の番号（第16章／第17章／第23章／第26章 など）に合わせてください。
